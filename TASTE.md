# TASTE.md — Evolving Preferences (Codex Sidekick iOS)

This file captures the aesthetic and product taste for the iPhone sidekick.
These preferences evolve, but they are the default bar.

---

## Philosophy

### Translate Codex, Don’t Clone iOS
The goal is not “a nice SwiftUI app.” The goal is Codex translated honestly to
phone form. Favor desktop-Codex sensibility over default Apple styling.

### Native, Not Webby
Use native interaction, scrolling, text input, and system ergonomics. But do
not let stock iOS styling dominate the product identity.

### Dense, Calm, Direct
The UI should feel efficient and focused. Prefer density over airiness, calm
surfaces over flashy decoration, and clear status over ornamental motion.

---

## Visual Direction

- Monochrome by default
- Status color only where meaning changes
- No gratuitous liquid glass
- No purple drift, neon accents, or Fawx palette borrowing
- Minimal chrome, but not sparse
- Terminal-adjacent typography and direct copy

---

## Layout

- Use the full device screen naturally.
- No letterboxing.
- No oversized hero padding once the app is in its working state.
- Prioritize useful rows above the fold.
- Compact controls should still feel deliberate, not cramped.

---

## Interaction

- Defaults should be obvious.
- Errors should tell the user exactly what to fix.
- Pairing should feel like choosing a trust path: local, tailnet, manual.
- Thread reading should bias toward recent activity, not top-of-history nostalgia.

---

## What to Avoid

- playful consumer-app polish
- oversized cards for low-information content
- decorative blur as a substitute for hierarchy
- whimsical copy
- “AI app” visual clichés
